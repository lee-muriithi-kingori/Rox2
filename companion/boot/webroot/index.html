Rox-Boot

v1.0 · companion to Rox2

I cannot fake the hardware. Rox-Boot hides kernel-level signals (cmdline, props, error markers). It does not defeat hardware-backed Play Integrity attestation. For STRONG integrity you must run TrickyStore with a keybox from your own device.

What gets cleaned at boot

ro.boot.flash.locked=1
ro.boot.verifiedbootstate=green
ro.boot.vbmeta.device_state=locked
ro.secureboot.lockstate=locked
sys.oem_unlock_allowed=0
ro.boot.mode=boot

Set in post-fs-data.sh via Magisk resetprop, before Zygote.

What gets scrubbed from /proc/cmdline

androidboot.unlocked=stripped
androidboot.verifiedbootstate=reset to green
androidboot.verifier=reset to locked

When the kernel allows bind-mounting a single /proc/cmdline file, Rox-Boot replaces the content. When it does not, the upstream cmdline remains visible — this is logged.

What I do not touch

  • Keymaster attestation. Whatever is in your device's TEE is what it is. Switching this would require a keybox from your own device (TrickyStore) — that path is documented in the Rox2 README.
  • vbmeta digest / size. I will not fake these. Stock firmware has device-specific values; a constant value across all users is itself a fingerprint.
  • Kernel cmdline if the kernel refuses bind-mount. I surface this in the log so you know it happened, not silently drop it.
